# Port 8000 "address already in use"

That message means **something is already running on port 8000** (often an earlier uvicorn).

## Option A — Use the server that’s already running

If the app is already running, you don’t need to start it again. Just run:

```bash
curl -s "http://127.0.0.1:8000/api/status"
```

Or open **http://127.0.0.1:8000** in your browser.

## Option B — Free port 8000 and start a new server

1. **Find what’s using port 8000:**
   ```bash
   lsof -i :8000
   ```
   Note the **PID** (first number in the right column).

2. **Stop that process** (replace `PID` with the number you saw):
   ```bash
   kill PID
   ```
   Example: `kill 32569`

3. **Start the app again:**
   ```bash
   cd /Users/mathewsposta/matsposta
   source venv/bin/activate
   python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
   ```

## Option C — Use a different port

```bash
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8001
```

Then open **http://127.0.0.1:8001** and use `http://127.0.0.1:8001/api/status` for curl.
